**版本**：v1.0 (MVP)
**建立日期**：2026-02-17
**設計師**：AI UI/UX Designer

---

## 一、設計策略

### 1.1 設計理念
**「簡約溫暖・高效直覺」**

結合 Apple 的簡約設計哲學與 LINE 的直覺操作體驗，打造一套門店人員在打烊疲勞狀態下仍能快速、流暢操作的營運管理系統。視覺上延續「阿爸的芋圓」品牌的溫暖大地色系，在專業工具的高效與品牌溫度之間取得平衡。

### 1.2 設計原則
1. **效率優先**：數字輸入是每日最大量的操作，必須極致流暢
2. **減少認知負擔**：品項分組清晰、操作路徑最短、重要資訊一目了然
3. **容錯友善**：輸入錯誤可快速修正，提交前可預覽確認
4. **視覺層次分明**：用色彩區分不同分類，用大小區分資訊優先級
5. **單手可操作**：所有核心操作都在拇指觸及範圍內

## 五、組件設計

### 5.1 數字輸入元件（核心元件 — 高頻使用）

這是系統最重要的元件，每天門店人員需要大量輸入數字。

#### 方案：「點擊即聚焦」數字輸入行

```
┌─────────────────────────────────────────┐
│ 🟫 配料類                                │ ← 分組標題（粘性置頂）
├─────────────────────────────────────────┤
│                                         │
│  紅豆           架上 [ 0.1 ] 盒          │ ← 點擊數字框聚焦
│  (期效7天)      庫存 [ 3/5 ] ▸ 日期欄     │
│                 倒掉 [ 0.5 ]             │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │ ← 分隔線
│  綠豆           架上 [ 0.1 ] 盒          │
│  (期效7天)      庫存 [     ]             │
│                 倒掉 [     ]             │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │
│  花生           架上 [   1 ] 盒          │
│  (期效7天)      庫存 [ 3/1 ] ▸ 09        │
│                 倒掉 [     ]             │
│                                         │
└─────────────────────────────────────────┘
```

**設計細節：**
- 輸入框使用 `inputmode="decimal"` 自動彈出數字鍵盤（含小數點）
- 點擊輸入框時全選現有數字，方便直接覆蓋輸入
- 按下 Enter 或點擊下一個欄位自動跳轉到下一個輸入框（Tab 流）
- 輸入框寬度固定 72px，右對齊數字，單位文字緊跟其後
- 已填入的欄位背景變淺綠 `#E8F5E9`，未填入保持 `#FAF8F6`
- 數字字體加大 20px，確保在疲勞狀態下也能清楚閱讀

#### 快速輸入流程
```
點擊第一個輸入框
    │
    ▼
數字鍵盤彈出（系統原生）
    │
    ▼
輸入數字 → 按「下一步/Enter」
    │
    ▼
自動跳至同行下一個欄位（架上→庫存→倒掉）
    │
    ▼
同品項填完 → 自動跳至下一品項的第一個欄位
    │
    ▼
持續輸入直到完成
```

## 七、交互規範

### 7.1 數字輸入交互流程（核心體驗）

```
Step 1: 用戶點擊輸入框
        → 輸入框高亮（品牌色邊框）
        → 系統彈出數字鍵盤（inputmode="decimal"）
        → 現有數字全選（方便覆蓋）

Step 2: 用戶輸入數字
        → 即時顯示在輸入框中
        → 右對齊，單位跟隨

Step 3: 用戶按 Enter / 點擊下一欄
        → 自動跳至下一個輸入框
        → 跳轉順序：架上 → 庫存 → 倒掉 → 下一品項架上
        → 已完成欄位背景變淺綠

Step 4: 完成一個分組
        → 分組標題顯示「4/4 已完成」✅
        → 自動滑動到下一分組
```

### 7.2 頁面轉場動畫
| 動作 | 動畫 |
|------|------|
| 進入頁面 | 從右滑入 (300ms, ease-out) |
| 返回 | 從左滑出 (250ms, ease-in) |
| 底部 Sheet | 從底部滑入 (350ms, spring) |
| Toast 提示 | 從頂部滑入 + 淡入 (200ms) |
| 提交成功 | 卡片縮小 + 打勾動畫 (400ms) |

### 7.3 手勢操作
| 手勢 | 動作 |
|------|------|
| 下拉 | 重新整理頁面資料 |
| 左滑品項行 | 顯示「備註」按鈕 |
| 長按品項 | 顯示該品項歷史數據（近7天） |

### 7.4 錯誤處理
| 場景 | 處理方式 |
|------|----------|
| 網路斷線 | 頂部出現紅色橫幅「網路已斷開，資料將暫存本地」 |
| 提交失敗 | Toast 提示 + 自動重試 + 保留已填資料 |
| 未填必填欄位 | 滑動至未填欄位，紅色邊框閃爍 |
| 數值異常（如叫貨量超大） | 彈出確認對話框「確定叫貨 100 袋芋圓嗎？」 |
